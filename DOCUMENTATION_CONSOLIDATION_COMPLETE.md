### The 4-File Approach

**Principle:** Every piece of documentation should have ONE clear home.

**Decision Tree:**
```
Is it about using the game?
  → MASTER_GUIDE.md

Is it about system architecture or operations?
  → MASTER_TECHNICAL.md

Is it about a specific feature implementation?
  → MASTER_FEATURES.md
```

**When adding new content:**
1. Determine which MASTER file it belongs to
